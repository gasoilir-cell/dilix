## ۱. خلاصه اجرایی

Dilix یک **بستر واسط جهانی (Global B2B2C Orchestration Platform)** است؛ نه یک شرکت مالی، بیمه‌ای یا مخابراتی. Dilix خودش وجه نگه نمی‌دارد، بیمه‌نامه صادر نمی‌کند و بارنامه رسمی صادر نمی‌کند. بلکه **ارائه‌دهندگانِ دارای مجوز** (بانک‌ها، شرکت‌های بیمه، شرکت‌های حمل‌ونقل، اپراتورها) را از طریق یک **Provider Adapter Framework** استاندارد به کاربران وصل می‌کند.

موتور جذب کاربر، یک **پوسته‌ی تعاملی (Engagement Shell)** شامل پیام‌رسان، شبکه اجتماعی، نقشه‌ی سه‌بعدی کره‌زمین و دستیار هوش مصنوعی است؛ و یک **Growth & Incentives Engine** قانونی (رفرال چندلایه‌ی محدود + revenue-sharing + marketplace کم‌کارمزد).

- **بازار اول:** ایران (شرکای آماده: بانک سامان، بیمه البرز، سازمان حمل‌ونقل).
- **توسعه:** روسیه، عمان، ترکیه (قراردادهای در جریان) → جهانی.
- **زیرساخت:** Hybrid Multi-Region (ریجن داخلی ایران + ریجن بین‌المللی) به‌دلیل تحریم و Data Residency.

## ۳. اصل ساخت: «طراحی جامع، اجرای لایه‌ای»

ما **مدل‌ها و contractهای هر ۱۲ ماژول را جامع طراحی می‌کنیم**، اما **عمیق فقط لایه ۰، ۱ و ۲ را** می‌سازیم. بقیه با همان Adapter Framework به‌موقع اضافه می‌شوند.

```
لایه ۰ — Platform Core
  Earth ID · Auth/MFA · RBAC/ABAC · Event Backbone · Provider Adapter · i18n · AI Gateway · Notification

لایه ۱ — Engagement Shell  (موتور جذب کاربر)
  Messenger · Social · 3D Earth Discovery · AI Assistant · Growth & Incentives

لایه ۲ — Vertical اول: Freight + Insurance + Payments
  اعلام بار · تطبیق راننده · تأییدیه‌های زنجیره‌ای · GPS · بارنامه (راهداری) · بیمه (البرز) · تسویه (سامان)

لایه ۳ — Verticalهای بعدی (Adapter-only)
  مالی کامل · مخابرات/eSIM · Service Marketplace · Open API Marketplace
```

## ۵. واقعیت‌های اجرایی «ایران-اول + جهانی»

| موضوع | چالش | راهکار معماری |
|---|---|---|
| توزیع اپ | محدودیت Play/App Store برای ایران | Cafe Bazaar + Myket + APK مستقیم + PWA؛ Play/App Store برای جهانی |
| زیرساخت | تحریم AWS/GCP برای ایران | Hybrid: ابر داخلی (آروان و…) برای ایران + ریجن بین‌المللی |
| نقشه | محدودیت Google Maps | CesiumJS/Mapbox برای کره‌ی 3D + نشان/بلد برای داخل |
| Push | ناپایداری FCM در ایران | WebSocket + push بومی به‌عنوان fallback |
| زبان | چندزبانه + RTL | i18n از روز اول (fa, en, ru, ar, tr) |
| پرداخت | شاپرک/شتاب داخلی vs بین‌المللی | Payment Adapter چندگانه (سامان داخلی + درگاه بین‌المللی) |
